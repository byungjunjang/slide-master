# AGENTS.md — Codex execution rules for slide-master

This repo is **dual-host**: Claude Code and Codex. [`CLAUDE.md`](CLAUDE.md) is the
SSOT for project pointers and standing conventions (route boundaries, Pretendard
font lock, `docs/rules/` style rules, markdown language consistency, compatibility
boundary) — **read it**. [`.claude/skills/ppt-master/SKILL.md`](.claude/skills/ppt-master/SKILL.md)
is the workflow authority. The rules below add Codex-side enforcement on top; they
restate discipline, they never override SKILL.md.

## Skill source (generated pointer stubs)

- **`.claude/skills/` is canonical** (`ppt-master`, `ppt-template-fill`,
  `diagram-design`, `codex-image`). **`.codex/skills/` holds
  generated discovery stubs** — same `name`/`description` frontmatter plus a
  redirect body — regenerated by
  `python3 .claude/skills/ppt-master/scripts/sync_codex_stubs.py`.
- **Never hand-edit `.codex/skills/`.** Edit the `.claude` tree, then re-run the
  sync script. `preflight.py` hard-fails when stubs are stale.
- When a stub triggers, read the canonical `.claude/skills/<name>/SKILL.md` **in
  full** and execute it as a runtime procedure — not as background documentation.
- Path resolution: `${SKILL_DIR}` and bare relative paths (`references/...`,
  `workflows/...`, `scripts/...`, `templates/...`) resolve against the canonical
  skill directory. Run all commands from the repository root. If `python3` is
  missing (common on Windows), rerun the same command with `python`.

## PPT requests — MANDATORY procedure

When the user asks for a presentation ("PPT 만들어줘", "make slides", "生成PPT",
a deck from a document/topic/template):

1. **Route deterministically** via
   [`workflows/routing.md`](.claude/skills/ppt-master/workflows/routing.md). The
   hard boundaries (raw-PPTX → `ppt-template-fill`; 1:1 → `beautify-pptx`;
   reusable template → `create-template`; finished-PPTX enhancement →
   `native-enhance-pptx`) are deterministic — do not improvise routing or ask
   the user to choose around a rule.
2. **Execute SKILL.md step by step.** Do NOT summarize it and improvise.
   **Forbidden:** hand-building a PPTX outside `svg_to_pptx.py`, generating SVG
   pages with a batch script, delegating SVG generation to sub-agents, batching
   pages in groups, or substituting placeholder/solid-color images for planned
   ones. If a step's tool is unavailable, follow
   [`workflows/failure-recovery.md`](.claude/skills/ppt-master/workflows/failure-recovery.md);
   never silently fall back.
3. **Preflight first:** after Step 2 project init, run
   `python3 .claude/skills/ppt-master/scripts/preflight.py` (add
   `--needs-images` for AI-image decks). If it FAILs, stop and fix the
   environment before continuing.
4. **⛔ BLOCKING gates are hard stops.** The Step 4 Strategist confirmation
   waits for explicit user confirmation — never decide on the user's behalf.
   Use the Confirm UI launch flow from SKILL.md; the staged chat fallback is the
   only alternative when the page cannot open.
5. **Spec gate:** immediately after writing `design_spec.md` + `spec_lock.md`,
   run `validate_spec.py <project_path>` and fix every error before generating
   any SVG.
6. **Images:** honor the confirmed image source (`result.json.image_usage`).
   The default `codex` AI path maps to **Codex's built-in `image_gen` tool**
   (a legacy `host-native` confirmation reads as `auto`) — save outputs to the
   exact `images/<filename>` targets from §VIII. On
   acquisition failure: retry once → mark `Needs-Manual` → report and continue
   (failure-recovery contract) — never a silent substitute image.
7. **Quality gates:** run `svg_quality_checker.py` on page 1 before drawing
   page 2, and on the whole project after the last page. Fix every `error`
   before proceeding.
8. **Post-processing discipline:** run as separate calls, in order —
   `total_md_split.py` (only when speaker notes were requested) →
   `finalize_svg.py` → `svg_to_pptx.py`. Never substitute `cp` for finalize;
   never use `-s final` for a release export.
9. **Completion gate:** before declaring the deck done, run
   `python3 .claude/skills/ppt-master/scripts/verify_deck.py <project_path>`
   and require exit 0. If it fails, fix the deck — do not declare completion.

## Non-negotiables (SSOT: CLAUDE.md / SKILL.md)

Serial pipeline, no cross-phase bundling · spec_lock re-read before every page ·
Pretendard font lock · `docs/rules/` style rules for any repo edit · workflow
package, not an app scaffold (no `tests/`, no `.worktrees/`).
