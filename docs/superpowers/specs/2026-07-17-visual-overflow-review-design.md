## 2. 검출 — `svg_quality_checker.py` 지오메트리 확장 (§3.1)

`check_file`에 새 검사 `_check_text_geometry()`를 추가한다. **template_mode에서는 스킵** ({{PLACEHOLDER}} 토큰 텍스트 오탐 방지).

### 2.1 폭 추정기

모듈 상수:

- CJK(한중일 통합·한글·가나·CJK 구두점·전각 형태·`—`·`…`) = **1.0 × font-size**
- Latin/숫자/기타 ASCII = **0.55 × font-size**
- 공백 = **0.25 × font-size**
- `letter-spacing` 존재 시 `(글자수−1) × letter-spacing` 가산
- **보수 마진 0.95**: `추정폭 × 0.95`가 경계를 넘어야 A류 error (과대추정 오탐 방지). 코퍼스 최소 침범(+54px, P06)도 이 마진에서 검출됨을 산수로 확인.

폰트 메트릭 정밀화로 과설계하지 말 것 — 근사 계수는 상수 튜닝으로 충분하다(§1.1 코퍼스 8/8 재현이 증거).

### 2.2 줄 단위 bbox

각 `<text>`에 대해 직접 텍스트(단일 줄) 또는 line-tspan 스택(각 줄)을 순회하며:

- 수평: `text-anchor`(start/middle/end) 반영 `[x_start, x_end]`
- 수직: baseline = `y + Σdy` 누적, 줄 박스 = `[baseline − 0.8×fs, baseline + 0.2×fs]`

### 2.3 A류 error

1. **캔버스 이탈**: 줄 박스가 viewBox 밖으로 나감 (좌/우/상/하).
2. **top-level `<g>` bbox 침범**: 줄 박스가 자기 그룹이 아닌 top-level `<g>`의 bbox와 교차. 장애물 bbox는 그 그룹의 rect/image/circle/ellipse/line/polygon/text/use 합집합 (**path는 제외** — 한계로 문서화; 이 덱 부류에서 장식 path는 드묾).
3. 푸터/이웃 세로 침범은 2와 같은 교차 판정 하나로 처리 (별도 메커니즘 없음).

### 2.4 오탐 예외 규칙 (§6-2 수용 기준의 대상)

1. **같은 `<g>` 내부** 요소 간 교차는 판정하지 않음 (예: P07 사이드바 사진 위 캡션).
2. **저투명 요소 제외**: `fill-opacity` 또는 `opacity` ≤ **0.15**인 요소는 장애물이 아님 (P03/P11 display numeral 워터마크).
3. **배경/hero 제외**: 캔버스 면적의 ≥ **85%**를 덮는 rect/image는 장애물이 아님 (P01/05/09/12/16/19/24 풀블리드 hero+스크림, 전면 흰 배경 rect).

### 2.5 B류 warning

spec_lock `typography.body`가 필요 — 없으면 B 검사는 스킵 (드리프트 검사의 "lock 없으면 침묵" 정책과 일치). 대상: **font-size ≥ body**이고 `text-anchor`가 start(기본)인 **다줄 line-tspan 스택**.

- **가용 폭** = `min(캔버스폭 − 캔버스폭×5%, 블록과 세로로 겹치는 가장 가까운 우측 장애물의 좌단) − x_start`. (5% ≈ 1280 캔버스의 64px 마진 — 코퍼스 산수와 일치. 장애물 판정은 §2.4 예외 적용.)
- **B-1 (불필요 줄바꿈)**: 전체 문장(줄들을 공백 결합) 추정폭 ≤ 가용 폭인데 감겨 있음 → warning "fits on one line — unwrap".
- **B-2 (카피 과길이 표면화)**: 추정폭 > 가용 폭인 경우 중,
  - **리드역(font-size > body)**: 다줄이면 무조건 warning (사용자 규범: 한 문장 리드의 두 줄 감김 자체가 결함),
  - **body 크기(== body)**: **한 문장**(문말 부호가 끝에만 있음) **+ 정확히 2줄**인 단문 블록만 warning.
  - warning 문구는 수선 방향 제시: "over-width copy — shorten (ladder ①) / redistribute zone (②) / verify balanced break (③, body prose only)".
- **침묵 대상**: 3줄+ 또는 다문장 프로즈 블록 (text 모드 덱의 정상 본문 — 오탐 방지).

### 2.6 spec_lock 상향 탐색

지오메트리 검사용 lock 탐색 헬퍼를 별도로 추가: `svg_path`에서 상향 **≤4 단계**까지 `spec_lock.md` 탐색. 동결 코퍼스(`<project>/backup/<ts>/svg_output/`)에서도 프로젝트 루트의 lock을 찾아 B 검사가 동작하게 한다. 기존 드리프트 검사의 2단계 탐색(`_get_spec_lock`)은 변경하지 않는다.

## 3. 예방 — 저작 규칙 (§3.2)

### 3.1 `references/executor-base.md`

"텍스트 폭 산수 의무" 신설 (§2.1 인근, 영문 — 시블링 언어 준수):

- 텍스트 블록을 그리기 **전** `est_width(문장, fs) ≤ 존 폭` 검증 — 근사식(검사기와 동일 상수)을 본문에 명기.
- 들어가면 **한 줄로** 그린다 — 감지 말 것 (B-1 근절).
- 초과 시 **수선 사다리** (핸드오프 §3.2 사용자 확정 순서):
  1. **카피 축약** — 리드/코어메시지 1순위. **Executor 권한** (브레인스토밍 확정 ⓑ): 의미 보존, 잘린 세부는 supporting 블록이 이미 보유하거나 그리로 이관, 사실 왜곡 금지.
  2. **존 재배분** — 충돌 상대가 움직일 수 있을 때 (패널/이미지 컬럼 폭 축소, 여백 재배분).
  3. **균형 줄바꿈** — **본문 프로즈 한정**: 어절/구 경계, 두 줄 길이 균형, 고아 단어 2행 금지. **리드/코어메시지 비허용**.
  4. **폰트 축소** — 원칙 금지 (기존 §1.0 body 한정·지역적·최대 −4px 최후수단 규칙 유지, 리드/타이틀 절대 불가).
- 세로 산수 병기: 마지막 baseline = `y + Σdy`, 블록 하단(+descent ≈ 0.2×fs)이 다음 요소 상단/캔버스 마진을 넘지 않을 것.

### 3.2 `references/strategist.md`

§6(§IX 저작 규칙 인근)에 **코어메시지/리드 카피 길이 예산** 추가:

- 가용 폭에서 역산한 글자수 상한: 전폭 레이아웃 `(1280 − 2×64) ÷ 40px ≈ 28자` → **안전 26자(CJK)**; 분할 레이아웃(사이드 패널/이미지) `≈ 16자`. 페이지가 사이드 비주얼을 계획하면 보수적으로 분할 기준 적용.
- 사용자 규범 명문화: **한 문장 리드/코어메시지가 두 줄로 감기는 것은 결함**. 코어메시지는 정의상 한 문장 스파인 — punchy가 규범, 세부는 supporting 블록으로.
