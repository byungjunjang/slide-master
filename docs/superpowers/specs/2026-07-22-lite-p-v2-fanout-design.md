## 2. 설계 — `scripts/consistency_check.py` (신규 독립 스크립트)

- 체커(4,545줄) 확장 대신 독립 스크립트: 크로스 페이지 검사는 팬아웃 워크플로 전용 관심사 — mandatory 게이트 의미론 무변경, 판정 미달 시 커밋 드롭이 깔끔. 읽기 전용 진단 도구라 판정과 무관하게 독립 머지 가능.
- 추출 로직 하나가 시드 생성과 검증 양쪽 담당(기준 불일치 원천 차단):
  - `--emit-seed --anchors P01,...` → 앵커에서 `anchor_vocab.json` 생성
  - 기본 모드 → 전 페이지를 시드 대역과 대조, 위반 시 exit 1
- 검사 항목: 크롬 수평선 y좌표(±2px) · 팔레트 밖 HEX · 이질 폰트 사이즈 · 마진 이탈. 리포트 형식은 checker의 error/warning 라인 관례를 따른다.
- 허용 오차 캘리브레이션: 기존 순차 벤치 덱(`20260721_bench_semis_roundA`)이 자기 검사를 통과하도록 조정 — 순차 덱 = 일관성 정답지.

### 7.2 기전 (왜 미승인가)

- fork 부재(환경 §0 실증)로 각 팬아웃 GP 서브에이전트가 콜드 컨텍스트를 부트스트랩(SKILL+참조+spec_lock+앵커 정독)해야 하고, 웨이브 벽시계는 **가장 느린 서브에이전트**가 게이트한다(24p 13m13s / 12p 9m29s).
- 묶음(2–3장)이 부트스트랩을 장당 1/3로 암탈하고 ref-diet가 참조를 줄였으나, 앵커 순차 구간(~8–11.5분) + 웨이브 갭 + 메인 일관성 게이트의 합이 순수 순차 저작을 넘어섰다. v1 실패 원인(장당 부트스트랩)이 크기만 줄어 잔존.
- fork(부트스트랩 ≈ 0, 컨텍스트 상속)가 전제였고 그것이 이 환경에 없다 — 차선책으로는 이 덱 크기에서 시간 이득 미실현.

### 7.3 캐비앗 (정직 고지)

- **n=1·에이전트 편차**: 이전 시리즈 교훈(콜드 A/B) — 동일 암 두 콜드 에이전트 편차(cold-A 22m45s vs cold-B 15m42s)가 큼. 12p 비교는 이전 세션 cold-A 재사용이라 덜 통제됨. **판정의 무게는 24p 동일세션 쌍**에 둔다. 총량 % 주장 금지(방향만).
- **consistency_check 알려진 한계**: left_margin 검사가 아키타입 미구분(구분자 등 의도적 다른 마진을 경고 발생). 경고 레벨이라 0-error 판정 불변. 차기 개선 후보(아키타입별 마진 밴드).

### 7.4 결정 (§5 사전등록 기준 실행)

- 시간 축 미달 → **v2 워크플로 커밋(`parallel-execute.md` + SKILL 규칙 6/7 예외 + 레지스트리) main 미머지.** v1 전례대로 브랜치 `lite-p-v2`를 실험 기록으로 보존(삭제 안 함).
- **`consistency_check.py`는 읽기 전용 독립 진단 도구** — 순차 덱에도 유효(정답지 2벌 캘리브레이션 0 error). 판정과 무관하게 선택적 독립 머지 가능(사용자 결정). 단, 워크플로 미머지 시 docstring의 `parallel-execute.md` 참조는 유지(브랜치 보존이라 링크 유효).
- 설계·계획·본 기록 문서는 유지(실험 원장).
